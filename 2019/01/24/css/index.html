CSS小结

发表于 2019-01-24 | 分类于 CSS

关于浏览器前缀，交给autofixer去解决，开发者或许应该把更多的精神放在创造上。

正如一开始所说，前端开发者的创造能力应该在视觉化方面展现的更加特殊，
那CSS3就是这么一柄利剑
幸好，CSS不像JavaScript一样那么多变化!
So, 快乐就完事儿了

  • 线性渐变 linear-grident

    • 当前很多设计师都会用到的一种艺术，或许在以前需要使用背景图。
    • 但是现在线性渐变就可以，只要你的分辨率不是太次就很清晰，放大也不会糊掉

  • Lea Verou’s DRY

    • 在整本书中，所见极多的就是Lea Verou的DRY(don’t repeat yourself)，但Lea Verou所说不是所谓的“工匠精神”，而是从维护者的角度出发的正确思想。
    • 尽量少的代码：如果一个元素的样式每次维护时需要更改6条属性，那么就存在1/6的bug隐患，因为我们很可能会忘记更改其中一个，那么这个时候尽量少的代码是最有效的降低这种问题的；
    • 代码易维护和代码量少不可兼得：这里倾向于维护自定义较强的样式，这里比较推荐拆分的写法。

  • currentColor 和 inherit

    • currentColor(当前颜色)，CSS中第一个变量，没有固定值，取的是当前文本的颜色，或许它应该更换一个名字，比如：currentTextColor
    • inherit(继承)，相对于currentColor而言inherit的作用域极其广，可以用在任何 CSS 属性中，继承自己父级的样式。
  • Lea Verou && 响应式网页设计
    • 响应式page，不单单是只有media query，还有更多的解决方案，但Lea Verou认为media query是作为最后一种解决方案 — 断点；
    • 百分比布局和视口单位的合理使用
    • 在较大分辨率下想要获取固定宽度时，max-width要优于width，所以你的width <= max-width
    • 在一些混合式的布局页面中，长宽未知却又需要继续布局，那么别犹豫了calc，让浏览器帮你计算。
