CSS小结

发表于 2019-01-24 | 分类于 CSS

关于浏览器前缀，交给autofixer去解决，开发者或许应该把更多的精神放在创造上。

正如一开始所说，前端开发者的创造能力应该在视觉化方面展现的更加特殊，
那CSS3就是这么一柄利剑
幸好，CSS不像JavaScript一样那么多变化!
So, 快乐就完事儿了

线性渐变 linear-grident

  • 当前很多设计师都会用到的一种艺术，或许在以前需要使用背景图。
  • 但是现在线性渐变就可以，只要你的分辨率不是太次就很清晰，放大也不会糊掉

Lea Verou’s DRY

  • 在整本书中，所见极多的就是Lea Verou的DRY(don’t repeat yourself)，但Lea Verou所说不是所谓的“工匠精神”，而是从维护者的角度出发的正确思想。
  • 尽量少的代码：如果一个元素的样式每次维护时需要更改6条属性，那么就存在1/6的bug隐患，因为我们很可能会忘记更改其中一个，那么这个时候尽量少的代码是最有效的降低这种问题的；
  • 代码易维护和代码量少不可兼得：这里倾向于维护自定义较强的样式，这里比较推荐拆分的写法。

currentColor 和 inherit

  • currentColor(当前颜色)，CSS中第一个变量，没有固定值，取的是当前文本的颜色，或许它应该更换一个名字，比如：currentTextColor
  • inherit(继承)，相对于currentColor而言inherit的作用域极其广，可以用在任何 CSS 属性中，继承自己父级的样式。

Lea Verou && 响应式网页设计

  • 响应式page，不单单是只有media query，还有更多的解决方案，但Lea Verou认为media query是作为最后一种解决方案 — 断点；
  • 百分比布局和视口单位的合理使用
  • 在较大分辨率下想要获取固定宽度时，max-width要优于width，所以你的width <= max-width
  • 在一些混合式的布局页面中，长宽未知却又需要继续布局，那么别犹豫了calc，让浏览器帮你计算。

或许有些难受的“Java写的admin项目”

  • 首先声明并非是在鄙视Java同学，只是指大多数，毕竟CSS对于Java来说，确实很头疼。
  • 首当其冲的float，一般来说闭合浮动对于广大前端来讲都是一件稀松平常的事情，但是在你后台项目中真的很多未闭合，如果你需要帮你的Java同学调试样式，那么请做好最大的耐心，这可能会让你有些抓狂。
  • 再者postion，在不知情的情况下，多往上看几层，或许你会发现很棒的事情，哦嚯，具体是什么深入进去就明白， 不得不说这挺锻炼人修改CSS的能力

layout的修炼

  • 不要问，问就是flex，如果你还不知道flex，那么廖雪峰老师的flex教程会打开你的一扇门，至于怎么去创造那就是诸君脑洞的事情了。
  • 被flex格式化的所谓block，其实并不能完全意义上说是块级元素，应该成为flex块
  • justify-content: space-around的边距实际上和calc计算值一样
  • flex + position 实际上被定位的元素不在flex布局的范围内，如果真的需要，那就给多套一层

适当的动画

  • 开始之前或许我们应该聊一聊性能开销，reflow & repaint
    • reflow 回流：当修改DOM元素的width、height、padding、margin等会影响到重新计算布局的属性时，触发回流，这是一项相当累的脏活，具体如何累，期待一下吧。
    • repaint 重绘： 当修改DOM元素颜色、背景色等不会影响计算布局时会触发重绘，成本略低于回流

  • 适当的动画能后有效的避免回流，比如transform

  • 纵然负边距也能达到同样的效果，但是transform触发回流的成本相对更低。

  • 当然，同样存在成本较低的方案，GPU加速

  • 适当的动画能让你的交互更加平滑，比如平滑的旋转90deg
