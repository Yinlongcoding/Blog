3.关于栅栏布局

概念

栅栏布局是一个响应式的，随着设备或者容器宽度适当的将容器扩展到12列(24列)。

实现方案与区别
  • 使用box-sizing: border-box，使得padding和border宽度不行影响到容器大小;
  • 使用媒体查询，做到大屏幕使用固定宽度/高度， 响应式中则是百分比
  • bootstrap4使用了flex布局来做的栅栏
  • bootstrap3和ElementUi则使用float来实现
为什么要选择使用浮动方案来实现

浮动兼容性比flexbox更加友好，在更加低版本的浏览器上可以保证布局不会混乱

移动端布局方案

1.说一下rem作用

rem是一个长度单位，是以<html>节点字体大小作为基准值进行长度计算，常用在移动端屏幕适配，通过等比缩放的方式达到设计图要求的效果

2.说一下有哪些值

Value 可能值 描述
width 固定像素(正整数)/device-width关键词 定义viewport（视口）的宽度
height 固定像素(正整数)/device-height关键词 定义viewport（视口）的高度
initial-scale 一个0.0 到10.0之间的正数 定义设备宽度与视口大小之间的缩放比率
maximum-scale 一个0.0 到10.0之间的正数 定义缩放的最大值，需要大于等于minimum-scale
minimum-scale 一个0.0 到10.0之间的正数 定义缩放的最小值，需要小于等于maximum-scale
user-scalable yes或者no 定义用户是否能放大或者缩小页面

#三问：什么是块级作用域

即词法作用域，在你写代码时将变量和块作用域写在哪里来决定，也就是词法作用域是静态的作用域，在你书写代码时就确定了。比较常见就是函数内部或者{}内部的代码块，能避免变量提升。

#四问：什么是变量提升

在声明变量的时候，都会在当前作用域顶层先声明变量且值为undefined，此后才进行赋值操作。函数声明的优先级高于变量申明的优先级，并且函数声明和函数定义的部分一起被提升。
