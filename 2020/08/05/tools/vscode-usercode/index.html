Visual Studio Code生活指北（一）

发表于 2020-08-05 | 分类于 VSCode


在？还在Ctrl + C 、V & Delete 修改前人代码来创建活动模版？
在？要不要考虑一下更快捷，更纯净的方式？

来都来了，先听段故事

    张三是个工程师，大学时期通过学习安装windows系列系统赢得美人归，惹得旁人一阵羡慕。
    但张三的生活并不如别人羡慕的那样，因为安装所用的Ghost(镜像)不是纯净版，里面混杂了各种流氓软件(xxx管家、xxx毒软件、xxx浏览器…)，所以张三每次花1个小时安装系统，2个小时去清理这些流氓软件，然鹅，女朋友还是总是抱怨说电脑卡，张三不能每次都重装系统啊，太低效了。
    于是，张三在github上找到了纯净Ghost，并在这个基础上，为女朋友量身添加了多个“大学女生必备装机软件”，赢得女朋友一阵夸，他骄傲的摸了摸自己全校CS竞赛的季军🏆。

故事讲完了，再说说我的小故事。某一次活动，我需要初始化活动文件(包含HTML,JS,CSS等)，我就去往期项目里随手抓了一个出来，Ctrl+C, Ctrl+V，然后修修改改，2分钟过去～然后，在开发过程中对引入的资源路径又是一顿修改，我猛然间觉得我和张三一开始很像，遂，我为什么不能也创造一个纯净的Ghost，通过宏命令的方式去创建活动模版。

查阅资料进度_99%

在查阅资料的过程中，我想起了大名鼎鼎的Emmit，一个!就可以将HTML模版输出，我就顺着这个线索去查，发现VSCode有用户代码片段这个功能，在短暂的查阅资料后，我就开始了创造过程…

一些参数说明：

  • prefix: 快捷指令，即要生成模版的宏命令
  • description: 宏指令描述，干啥的
  • ${Number} 首次鼠标闪烁的地方，Tab键依次跳转数字下一个位置
